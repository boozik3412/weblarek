# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

## Модели данных


#### Класс Products

Отвечает за управление каталогом товаров. Хранит список всех доступных товаров и предоставляет методы для работы с ними.

**Конструктор:** `constructor()` - инициализирует пустой массив товаров.

**Поля класса:**
- `_items: IProduct[] = []` - приватное поле для хранения массива товаров типа `IProduct`.

**Методы класса:**
- `setItems(items: IProduct[]): void` - устанавливает массив товаров. Принимает массив объектов типа `IProduct` и сохраняет его в `_items`.
- `getItems(): IProduct[]` - возвращает текущий массив товаров типа `IProduct[]`.
- `getItem(id: string): IProduct | undefined` - находит и возвращает товар по идентификатору. Параметр `id` типа `string`, возвращает `IProduct` или `undefined` если товар не найден.
- `addItem(item: IProduct): void` - добавляет новый товар в каталог. Принимает объект типа `IProduct`.

#### Класс Basket

Управляет корзиной покупок. Позволяет добавлять товары в корзину, удалять их, считать итоговую стоимость и количество товаров.

**Конструктор:** `constructor()` - инициализирует пустой массив товаров в корзине.

**Поля класса:**
- `_items: IProduct[] = []` - приватное поле для хранения товаров в корзине типа `IProduct[]`.

**Методы класса:**
- `getItems(): IProduct[]` - возвращает массив товаров, находящихся в корзине.
- `addItem(item: IProduct): void` - добавляет товар в корзину. Если товар уже есть в корзине, увеличивает его количество (`quantity`) на 1. Новый товар добавляется с `quantity: 1`. Принимает объект типа `IProduct`.
- `removeItem(itemId: string): void` - удаляет товар из корзины по его идентификатору. Параметр `itemId` типа `string`.
- `clear(): void` - полностью очищает корзину, удаляя все товары.
- `getTotal(): number` - вычисляет и возвращает общую стоимость всех товаров в корзине (с учетом количества). Возвращает число типа `number`.
- `getCount(): number` - возвращает общее количество товаров в корзине (с учетом повторений). Возвращает число типа `number`.
- `hasItem(itemId: string): boolean` - проверяет, содержится ли товар с заданным идентификатором в корзине. Параметр `itemId` типа `string`, возвращает `boolean`.

#### Класс Order

Представляет заказ и управляет его состоянием. Содержит информацию о товарах в заказе, данные покупателя и методы для изменения статуса и способа оплаты.

**Конструктор:** `constructor(buyer: IBuyer, items: IProduct[], total: number)` - инициализирует заказ с информацией о покупателе и товарах. Параметры: `buyer` типа `IBuyer` (содержит phone, email, address), `items` типа `IProduct[]`, `total` типа `number`.

**Поля класса:**
- `id: string = ""` - идентификатор заказа, устанавливается при отправке на сервер.
- `items: IProduct[] = []` - массив товаров в заказе типа `IProduct[]`.
- `total: number = 0` - общая стоимость заказа типа `number`.
- `paymentMethod: string = ""` - способ оплаты (тип `string`).
- `address: string = ""` - адрес доставки (тип `string`).
- `phone: string = ""` - телефон покупателя (тип `string`).
- `email: string = ""` - email покупателя (тип `string`).
- `status: string = ""` - статус заказа (тип `string`).

**Методы класса:**
- `setPayment(method: string): void` - устанавливает способ оплаты. Принимает строку `method` с названием способа оплаты.
- `getPayment(): string` - возвращает текущий способ оплаты типа `string`.
- `setStatus(status: string): void` - изменяет статус заказа. Принимает строку `status` с новым статусом.
- `getStatus(): string` - возвращает текущий статус заказа типа `string`.
- `getTotal(): number` - возвращает общую стоимость заказа типа `number`.
- `getItems(): IProduct[]` - возвращает массив товаров в заказе типа `IProduct[]`.
- `getBuyer()` - возвращает объект с данными покупателя (phone, email, address).

## Взаимодействие с API

#### Класс LarekAPI

Обеспечивает взаимодействие приложения с сервером. Использует базовый класс `Api` для выполнения HTTP-запросов. Реализует специфичные для приложения методы получения товаров и отправки заказов.

**Конструктор:** `constructor(private api: IApi)` - принимает объект типа `IApi` (интерфейс базового класса `Api`) для выполнения HTTP-запросов.

**Поля класса:**
- `api: IApi` - приватное поле для хранения экземпляра класса `Api`, используется для отправки запросов на сервер.

**Интерфейсы, используемые классом:**
- `IProductsResponse` - интерфейс для ответа сервера при запросе товаров. Содержит поле `items: IProduct[]` - массив товаров.
- `IOrderResponse` - интерфейс для ответа сервера при отправке заказа. Содержит поле `id: string` - идентификатор созданного заказа.

**Методы класса:**
- `getProducts(): Promise<IProduct[]>` - отправляет GET-запрос на эндпоинт `/product/` для получения списка товаров с сервера. Возвращает промис, который разрешается массивом товаров типа `IProduct[]`. Использует композицию с `Api` для выполнения запроса.
- `postOrder(order: any): Promise<IOrderResponse>` - отправляет POST-запрос на эндпоинт `/order/` для отправки заказа на сервер. Принимает объект заказа и возвращает промис с ответом типа `IOrderResponse`, содержащим идентификатор созданного заказа. Использует композицию с `Api` для выполнения запроса.

**Архитектурные решения:**
- LarekAPI использует функционал класса `Api` через **композицию** (а не наследование), что позволяет отделить логику работы с сервером от основных операций с API.
- Класс не содержит прямую интеграцию запросов в классы работы с данными (Products, Basket, Order), следуя принципу разделения ответственности в архитектуре MVP.
- Все HTTP-запросы проходят через базовый класс `Api`, который обрабатывает ошибки и форматирует ответы сервера.
